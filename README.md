# 📝 Офлайн-заметки с PWA и Push-уведомлениями

## О проекте

Веб-приложение для создания заметок и напоминаний, демонстрирующее современные технологии фронтенд-разработки:

- Полная офлайн-работа через Service Worker
- Установка на устройство как PWA (Web App Manifest)
- Реальное время через WebSocket (Socket.IO)
- Push-уведомления с возможностью откладывания
- Архитектура App Shell для мгновенной загрузки
- HTTPS для безопасного соединения

---

## Выполненные практические работы

### №13 — Service Worker

**Цель:** Обеспечить работу приложения без доступа к интернету.

**Реализовано:**
- Регистрация Service Worker при загрузке страницы
- Кэширование статических ресурсов (HTML, JS, манифест)
- Перехват fetch-запросов и возврат ответов из кэша при отсутствии сети
- Очистка старых кэшей при активации новой версии

**Результат:** Приложение полностью загружается и работает без интернета.

---

### №14 — Web App Manifest

**Цель:** Превратить веб-сайт в устанавливаемое PWA.

**Реализовано:**
- Создан файл `manifest.json` с метаданными
- Добавлены иконки 7 размеров (16×16, 32×32, 48×48, 64×64, 128×128, 256×256, 512×512)
- Настроены `theme_color`, `background_color`, `display: standalone`
- Добавлены мета-теги для iOS (apple-touch-icon, status-bar-style)

**Результат:** Приложение можно установить на рабочий стол, оно запускается в отдельном окне без адресной строки.

---

### №15 — HTTPS + App Shell

**Цель:** Обеспечить безопасное соединение и мгновенную загрузку интерфейса.

**Реализовано:**
- Настройка локального HTTPS через `mkcert` (сертификаты `localhost+2.pem` и `localhost+2-key.pem`)
- Архитектура App Shell: `index.html` содержит только каркас
- Динамическая подгрузка контента (`home.html`, `about.html`) через `fetch`
- Стратегии кэширования: Cache First для статики, Network First для контента

**Результат:** Каркас загружается мгновенно, контент подгружается асинхронно, соединение защищено.

---

### №16 — WebSocket + Push

**Цель:** Реализовать двустороннюю связь в реальном времени и push-уведомления.

**Реализовано:**
- Сервер на Node.js + Express + Socket.IO
- Генерация VAPID-ключей для push-уведомлений
- Клиент подключается к серверу через Socket.IO
- При добавлении заметки событие рассылается всем клиентам
- Push-уведомление отправляется всем подписанным пользователям
- Кнопки "Включить уведомления" и "Отключить уведомления"

**Результат:** При добавлении заметки все открытые вкладки получают уведомление мгновенно, push-уведомления приходят даже при закрытом браузере.

---

### №17 — Детализация Push (Напоминания)

**Цель:** Добавить гибкое управление напоминаниями.

**Реализовано:**
- Форма для создания заметок с выбором даты и времени
- Хранение заметок с полями: `id`, `text`, `reminder` (timestamp)
- Сервер планирует push-уведомления через `setTimeout`
- В уведомлении появляется кнопка "Отложить на 5 минут"
- При нажатии отправляется запрос на сервер (`/snooze`), таймер сбрасывается

**Результат:** Пользователь получает напоминание в выбранное время, может отложить его на 5 минут.

## Технологический стек

| Категория | Технологии |
|-----------|------------|
| **Frontend** | HTML5, CSS3 (Chota CSS-фреймворк), JavaScript (ES6+) |
| **Backend** | Node.js, Express.js, Socket.IO |
| **Push-уведомления** | web-push, VAPID-ключи |
| **PWA** | Service Worker, Web App Manifest, App Shell |
| **Хранение** | localStorage (заметки), Map (таймеры на сервере) |
| **Безопасность** | HTTPS (самоподписанный сертификат через mkcert) |
